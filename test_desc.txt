How the test file works
-----------------------
The notebook is never converted to a .py file. Instead, the test runner
reads the .ipynb JSON directly, concatenates every code cell, strips out
the lines that would load data or display matplotlib plots, and exec's
the rest into a namespace. The four core classes (TrafficDistribution,
Edge, Node, TrafficGraph) and the reconstruct_path function are then
pulled out of that namespace and tested normally with pytest.

A module-scoped fixture loads the full graph from the CSV files once and
shares it across every test that needs real data.

     haversine_distance(Buffalo, Rochester) must equal

3.3  test_haversine_less_than_road                               PASSED
     The straight-line distance between Buffalo and Rochester must
     be strictly less than 73 miles (the approximate I-90 driving
     distance). This is a spot-check that the heuristic is
     admissible for a known city pair.

3.4  test_admissibility_all_edges                                FAILED
     Iterates over every unique edge in the dataset and checks
     that the Haversine distance between the two endpoint cities
     is <= the road mileage listed in the CSV. This is the formal
     admissibility requirement: if the heuristic ever overestimates
     the true cost, A* can return suboptimal paths.

     Failure details:
       Edge    : New York City → Yonkers via I-87
       Road mi : 15.0  (from CSV)
       Haversine: 16.25 (computed from GPS coordinates)
       Problem : The CSV lists 15 miles, but the GPS coordinates
                 are 16.25 miles apart as the crow flies. A road
                 cannot be shorter than the straight-line distance.
       Fix     : Either increase the mileage to ~17 in the edges
                 CSV, or adjust the GPS coordinates of NYC/Yonkers
                 so the straight-line distance falls below 15.
